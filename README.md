# CDiskManager

一款用于定位、解释并处理 Windows C 盘空间占用的桌面工具，基于 **WinUI 3 / .NET 8** 构建，采用 MVVM 架构。

它的核心目标不是“盲目删除文件”，而是帮助用户先看清系统盘是否紧张，再按风险最小的路径处理：查看占用来源、清理缓存、定位大文件、识别重复文件，并在删除前给出确认和失败反馈。

## 功能

- **概览**：以 C 盘为中心展示使用率、可用/已用/总容量、文件系统、风险状态和下一步建议，并提供扫描、清理、大文件、重复文件入口。
- **空间扫描**：递归扫描所选分区，按占用大小排序展示文件夹/文件，提供 Top 占用摘要、面包屑下钻、当前目录搜索、文件/文件夹过滤和“在资源管理器中打开”。
- **垃圾清理**：检测并清理临时文件、Windows Update 缓存、预读取、缩略图、浏览器缓存、应用缓存、系统日志、传递优化、着色器缓存、回收站和 Docker 可回收项目，清理后展示释放空间、删除数量、失败数量和失败示例。
- **Docker 专清**：拆分 `docker system prune -af` 与 `docker volume prune -f`。未安装 Docker、Docker Desktop 未启动、daemon 不可访问、命令超时和“无可回收项”会给出不同提示。
- **大文件检测**：按可配置阈值查找大文件，展示合计占用和当前选择预计释放空间，支持批量选择、打开所在位置、删除，并反馈删除失败示例。
- **重复文件检测**：基于「大小 → 部分哈希 → 完整 SHA256」三级比对识别内容相同的文件，默认保留最早副本，并阻止用户删除某一组中的全部副本；扫描时显示当前路径，未勾选时删除按钮禁用。
- **分区建议**：分析 C 盘中的用户文件夹和其他分区余量，给出迁移到其他分区的建议，并明确这些建议不会自动移动文件。
- **缓存迁移**：识别 B站、QQ、微信、企业微信、网易云、浏览器和开发工具缓存，可一键迁移到 C 盘以外的目标盘并在原路径创建目录联接。高风险目录（聊天接收文件、Local Storage、IndexedDB 等）不会默认勾选。
- **设置**：主题切换（跟随系统/浅色/深色）、默认扫描分区、扫描阈值、删除到回收站开关、缓存迁移目标盘，配置持久化到本地并进行边界规范化。

## 安全设计

- 删除操作默认进入**回收站**，可在设置中切换为永久删除。
- 所有删除/清理操作均有确认对话框。
- 重复文件删除会强制每组至少保留一个文件，避免误删整组。
- 删除和清理会反馈失败数量及失败示例，避免“点了没反应”。
- 受保护的系统目录需要管理员权限，应用会在需要时提示「以管理员身份重启」。
- 系统级/高风险清理项不会默认勾选；Docker volume 单独列出并带有数据风险提示。
- 缓存迁移会要求关闭相关客户端。取消迁移会保留取消语义，失败项会保留原路径并展示失败示例。
- 设置中的异常主题、非法阈值和错误扫描路径会被规范化，避免影响后续扫描入口。

## 技术栈

- WinUI 3（Windows App SDK 1.5）
- .NET 8（`net8.0-windows10.0.22621.0`，x64）
- CommunityToolkit.Mvvm
- Microsoft.Extensions.DependencyInjection

## 构建与运行

```pwsh
dotnet build CDiskManager.sln -c Debug -p:Platform=x64
```

生成的可执行文件位于：

```
CDiskManager/bin/x64/Debug/net8.0-windows10.0.22621.0/win-x64/CDiskManager.exe
```

## 验证

本项目当前主要通过以下方式验证：

```pwsh
dotnet build CDiskManager.sln -c Debug -p:Platform=x64
dotnet test CDiskManager.sln -c Debug -p:Platform=x64 --no-restore
```

当前测试项目为 `CDiskManager.Tests`，会在临时目录中覆盖核心服务行为：

- 空间扫描能正确统计目录大小和文件数量。
- 空间扫描结果能同时包含子目录和当前目录直接文件。
- 重复文件检测能找出内容相同的文件组。
- 小规模重复文件扫描也会报告进度。
- 重复文件删除保护会阻止整组全选删除。
- 清理服务能区分成功删除、占用失败、缺失路径。
- Docker 输出解析和 Docker 不可用状态提示。
- 缓存迁移能移动目录并创建联接，且取消不会被吞成普通失败。
- 文件删除服务不会把不存在文件误报为删除成功。
- 设置规范化会修正非法主题、阈值和默认扫描路径。

此外，每轮功能改动后都会执行 Debug 应用启动冒烟检查，确认 WinUI 应用不会启动即退出。

## 项目结构

```
CDiskManager/
├── Models/        数据模型
├── Services/      扫描、清理、重复检测、分区分析、设置等服务
├── ViewModels/    各页面的视图模型（MVVM）
├── Views/         各页面的 XAML 与代码后置
├── Helpers/       转换器、原生互操作、管理员权限等辅助类
├── App.xaml(.cs)  应用入口与依赖注入配置
└── MainWindow     主窗口与导航
```
